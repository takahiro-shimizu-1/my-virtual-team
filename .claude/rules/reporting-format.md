# 結果の統合と報告

sub-agent の結果を返すときは、誰が何を出したか、次に何を読むべきかがすぐ分かる形でまとめる。

## 基本形式

```text
━━━ 稼働エージェント ━━━
主担当: {名前}（{部門} / {専門領域}）
サブ  : {名前}（必要な場合のみ）

要約:
- {今回の結論}

成果物:
- outputs/{file}
- outputs/{handoff}.json

確認事項:
- {確認が必要な点。なければ省略}

━━━ [DONE/WIP/REVIEW] [{部門} {名前}] {タスク内容} ━━━
```

## 統合ルール

- 単一 agent なら要約と成果物をそのまま返す
- 複数 agent の場合は重複を削り、担当単位ではなく結論単位で統合する
- 矛盾がある場合は一方を消さず、論点として併記する
- 次フェーズがある場合は handoff JSON の場所を必ず明示する
