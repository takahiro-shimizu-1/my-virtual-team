# 品質基準

## 基本ルール
- 成果物は必ず「誰に」「何のために」を明確にしてから作成する
- 結論・要点を先に述べ、詳細は後に続ける
- 1つの成果物は1つの目的に絞る

## フォーマット共通
- 見出し（##）で構造化する
- 箇条書きを活用し、視認性を確保する
- 長文は避け、1段落3行以内を目安にする

## 長さの目安
| 成果物 | 目安 |
|---|---|
| SNS投稿（X） | 140文字以内（日本語） |
| 要約・サマリー | 300〜500文字 |
| レポート・分析 | 1,000〜3,000文字 |
| 提案書 | 2,000〜5,000文字 |
| 要件定義書 | 必要に応じて（構造優先） |

## チェックリスト（全成果物共通）
- [ ] 誤字脱字がないか
- [ ] 事実と意見が区別されているか
- [ ] 出典・根拠が明記されているか（調査系）
- [ ] ターゲットに合った言葉遣いか
- [ ] ブランドガイドラインに沿っているか
- [ ] 結論が明確か

## コード成果物の追加チェック
- [ ] 動作確認済みか
- [ ] セキュリティ上の問題がないか
- [ ] エラーハンドリングが適切か
- [ ] コメントが必要な箇所に記載されているか
