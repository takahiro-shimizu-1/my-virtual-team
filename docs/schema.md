# my-virtual-team Schema

## Agent Frontmatter

すべての `agents/*.md` は以下の frontmatter を持つ。

```yaml
---
agent_id: string
department: string
keywords: [string, ...]
context_refs:
  always: [path, ...]
  on_demand: [path, ...]
  never: [path, ...]
context_budget: number
approval_policy: string
execution_mode: string
---
```

### `AGENTS_CLAUDE.md`

GitNexus 現行 indexer 互換のために frontmatter から生成する。

```yaml
# generated by npm run registry:build
generated_agents:
  - agent_id: "asahina-yu"
    name: "朝比奈 ユウ"
    role: "X を中心とした SNS 戦略の立案とコンテンツ作成を担当する。"
    society: "03-marketing"
    type: "local"
    keywords:
      - "SNS投稿"
      - "X運用"
```

### `.gitnexus/knowledge/`

GitNexus の `knowledge_dir` には curated mirror を渡す。

- `AGENTS_CLAUDE.md`
- `guidelines/`
- `docs/`
- `templates/`
- `.claude/rules/`
- `CLAUDE.md`
- `DESIGN_CONSTRAINTS.md`

これは GitNexus 互換のための生成レイヤーであり、SSOT ではない。

## Future Durable Store

- `tasks`
- `task_dependencies`

- `task_events`

- `task_attempts`
- `task_events`
- `task_outputs`
